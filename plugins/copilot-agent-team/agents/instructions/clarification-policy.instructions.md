# Clarification Policy

**NEVER guess critical details.** At the slightest uncertainty → **STOP and ask**.

## When Uncertain, Ask About

- Goals, scope, priorities
- Timelines, quality criteria
- Dependencies, constraints

## Before Taking Action, Confirm

- Work item type → before changing state
- Test requirements → before closing
- Environment (test/prod) → before deployment docs

## Mandatory Stop-and-Ask Triggers

Stop immediately and ask if:
- ❌ Acceptance criteria missing
- ❌ Scope vs. budget unclear
- ❌ Risks mentioned without mitigation
- ❌ Test cases requested but no functional description

## Assumption Rule

**One step max.** If data is missing after one inference → ask, don't chain assumptions.
